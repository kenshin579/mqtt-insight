# mqtt-insight 수동 E2E 체크리스트 (redesign)

릴리스/머지 전 실제 GUI로 확인하는 시나리오. 각 항목에 관련 요구사항 레지스트리 ID를 괄호로 병기했다
(`docs/superpowers/specs/2026-07-03-redesign-inventory.md` 참조). 프로토타입과 나란히 비교할 때는
`docs/design/first-redesign/MQTT Insight Redesign.dc.html`을 기준으로 삼는다.

## 준비

**원커맨드 (권장)** — 브로커 + retained 시드 + 앱 + 라이브 피드를 한 번에:

```bash
./run.sh          # 전부 실행
./run.sh down     # 전부 정리
./run.sh app      # 코드 수정 후 앱만 재빌드+재실행
./run.sh status   # 상태 확인
```

세부 제어는 `scripts/dev-env.sh` 참조 (`feed` 서브커맨드, `INTERVAL`/`DURATION` 환경변수).

<details><summary>수동 셋업 (스크립트 없이)</summary>

```bash
docker run -d --name mosq -p 1883:1883 eclipse-mosquitto:2 \
  sh -c "printf 'listener 1883\nallow_anonymous true\n' > /mosquitto/config/mosquitto.conf && exec mosquitto -c /mosquitto/config/mosquitto.conf"
```

앱 실행: `wails dev` (또는 `wails build -clean` 후 `open build/bin/mqtt-insight.app`)
</details>

메시지 발행용 CLI(별도 터미널):

```bash
docker exec mosq mosquitto_pub -t sensors/room1/temp -m 23.4 -r -q 1
docker exec mosq mosquitto_pub -t sensors/room1/hum -m '{"v":55,"unit":"%"}' -q 0
```

정리(모든 시나리오 완료 후): `docker rm -f mosq`

## 1. Welcome (A1)

- [ ] 최초 실행(프로필 0개) → Welcome 화면: 히어로 아이콘, 타이틀/서브, 3단계 카드(1: 연결 생성, 2: `#` mono span 포함 구독 안내, 3: 메시지 확인), CTA 버튼
- [ ] CTA 클릭 → 연결 모달 오픈
- [ ] 연결된 상태에서 타이틀바 `?` 클릭 → 가이드 오버레이로 Welcome 재표시, "돌아가기" 클릭 시 원래 화면(트리·메시지·발행 등 세션 상태)이 그대로 유지됨(F6) — 즉 연결이 끊기거나 트리/구독이 초기화되지 않아야 함
- [ ] 가이드 오버레이에서 Esc → 오버레이 닫힘 (F28)

## 2. Connection Home (A2)

- [ ] 프로필 1개 이상 && 미연결 상태로 진입(연결 해제 또는 앱 재시작) → Connection Home, 첫 프로필 자동 선택
- [ ] 좌측 목록에서 카드 클릭 → 선택 표시(파란 보더/배경) + 우측 상세 갱신, 이전 오류 배너 클리어
- [ ] 카드 더블클릭 → 즉시 연결 시도(연결 중 오버레이로 전환)
- [ ] "편집" → 연결 모달이 고급 탭으로 열리고 **전 필드가 프리필**됨(G1): 이름/버전/호스트/포트/전송/Client ID/Keep-alive/Clean session/Auto-reconnect/User/Pass, TLS 전송이면 CA 경로·시스템 CA·검증 생략, WS/WSS 전송이면 WebSocket 경로, LWT 섹션(topic/payload/QoS/retained) 전부
- [ ] "삭제" → 확인 다이얼로그 표시(F27), 취소 시 삭제되지 않음, 확인 시 목록에서 제거
- [ ] 마지막 프로필 삭제 → Welcome으로 전환
- [ ] 프로필 미선택 상태(목록은 있지만 selected 없음, 예: 방금 마지막 카드가 아닌 다른 이유로 unset) → `←` empty state(A14)

## 3. 연결 모달 (A4)

- [ ] 빠른(Quick) 탭: 호스트/포트/전송만 노출, 힌트 박스 표시
- [ ] 고급(Advanced) 탭: 프로필 이름/버전/호스트/포트/전송/Client ID/Keep-alive/Clean session/Auto-reconnect/User/Pass + 전송에 따라 TLS 또는 WS 섹션 + LWT 섹션이 노출/숨김
- [ ] 저장된 프로필 칩 목록 표시, "+ 새 연결" 칩 선택 시 폼이 기본값으로 리셋, 프로필 칩 클릭 시 해당 값으로 폼 채움
- [ ] 폼 필드를 직접 수정하면 선택된 칩이 해제됨(B47)
- [ ] 오류 배너 카피가 실패 사유별로 다르게 표시됨(F33/G19):
  - 잘못된 자격증명(인증 실패 브로커) → `errAuth`
  - TLS 인증서 오류 → `errTls`
  - 연결 거부(예: `nc -l 1999` 등으로 포트 막고 접속) → `errRefused`
  - 존재하지 않는 호스트(예: `nohost.invalid`) → `errUnknownHost`
  - 타임아웃(느리게 응답하는 호스트/방화벽 drop) → `errTimeout`
  - 그 외 → `errGeneric` + 원문 에러 병기
- [ ] 호스트/포트 입력창에서 **Enter** → 연결 시도(F28/C42)
- [ ] **Esc** → 모달 닫힘(F28/C42), 백드롭 클릭으로도 닫힘
- [ ] 연결 취소 후 모달이 닫히지 않고 유지됨(F29) — 오류 발생 시 폼 값이 사라지지 않아 재시도 가능한지 확인

## 4. 연결 중 오버레이 (A8)

- [ ] 응답이 느린 호스트(예: 방화벽에서 SYN drop되는 사설 IP)로 연결 시도 → 전면 dim + 스피너 + "연결 중" 라벨 + 취소 버튼 오버레이 표시
- [ ] 취소 클릭 → `CancelConnect` 호출, 오버레이 닫히고 연결 시도 중단, 앱이 정상 응답(C13)

## 5. 재연결 · 끊김 배너 (A9/A10)

브로커를 강제로 재시작해 재연결 사이클을 관찰한다:

```bash
docker restart mosq
```

- [ ] 연결된 상태에서 위 명령 실행 → 연결 바 아래 노란 재연결 배너 표시, 스피너 + "재연결 시도 중… (시도 N)" 카피, N이 1부터 순차 증가(C15~C17)
- [ ] 배너의 "지금 재시도" 버튼 클릭 → 즉시 재시도(시각적으로 확인 가능하면 확인)
- [ ] 배너의 "중단" 버튼 클릭 → `Disconnect` 호출, 재연결 중단되고 상태가 끊김으로 바뀜
- [ ] 브로커가 다시 응답 가능해지면 자동 재연결 성공 → 배너 사라지고 상태 점 초록으로 복귀, 트리/구독/메시지 등 기존 데이터가 보존됨(C18, 데이터 보존)
- [ ] 브로커를 계속 내려둔 채 auto-reconnect가 꺼진 프로필로 연결이 완전히 끊기면 → 빨간 끊김 배너(⚠ + 메시지 + "다시 연결" solid red 버튼), Main App 뷰 자체는 유지됨(데이터 보존)
- [ ] "다시 연결" 클릭 → 동일 프로필로 재연결 시도

## 6. 트리 패널 (A12, A15, B19~B27)

- [ ] 구독 0 && 토픽 0 상태에서 empty state(↯ 아이콘 ringpulse, "모든 토픽 구독 #" 버튼, 특정 토픽 입력)
- [ ] "모든 토픽 구독 #" 클릭 → `#` 구독 후 메시지 발행 시 트리에 계층 생성
- [ ] 특정 토픽 입력 후 **Enter** → 구독됨(F28/C42), 인풋 값 초기화
- [ ] 구독 칩 표시(패턴 또는 `pattern · qN`, QoS≠0일 때만 qN 노출), 칩 내부 ✕ 클릭 → 구독 해지
- [ ] 구독 중인 패턴에 매칭되는 리프는 정상 색상, 매칭되지 않는(해지된) 토픽의 리프는 dim 처리(C20/B26); 다시 구독하면 un-dim(F26)
- [ ] "+ 추가" 칩 → 인라인 추가 행 토글, 패턴 입력창에서 **Enter** → 구독(F28/C42), 중복 패턴은 무시하되 입력은 클리어됨(C19)
- [ ] 트리 필터 입력 → 전체 경로(fullTopic) 기준 대소문자 무시 substring 필터(F21), 매칭된 리프의 조상 브랜치도 함께 표시
- [ ] 구독이 1개 이상 존재하는 상태에서 1회성 힌트 카드(⋯ 아이콘 + 안내 문구) 표시(F17), ✕ 클릭 시 사라지고 앱 재시작 후에도 다시 나타나지 않음(config 영속)
- [ ] 트리 행 우클릭 또는 ⋯ 버튼 → 컨텍스트 메뉴 표시, 항목 4종 커버(D29~D31 라벨 기준): "이 토픽에 발행", "기록 켜기"/"기록 끄기"(토글), retained 리프에서만 "Retained 삭제" 노출
  - [ ] "이 토픽에 발행" → 발행 패널 topic이 채워지고 힌트 텍스트 표시(C22/C24)
  - [ ] "기록 켜기" → 리프 옆 ● 표시, 다시 "기록 끄기" → ● 사라짐
  - [ ] retained 리프에서 "Retained 삭제" → 재구독 시 해당 retained 메시지 미수신
  - [ ] 브랜치(비 리프) 노드는 기록 여부와 무관하게 ● 표시가 나타나지 않음(F31)
  - [ ] 메뉴 밖 클릭 또는 Esc로 닫힘(G14, ContextMenu 기존 동작 유지)
- [ ] 처음 기록을 켤 때 하단 중앙 토스트(기록 안내 카피) 표시, 6.5초 후 자동 소멸, 세션 재진입/재시작 후에는 다시 뜨지 않음(A7, config 영속)

## 7. 메시지 패널 (A13, B28~B31)

- [ ] 토픽 미선택 시 `←` empty state, 선택했지만 메시지 없음 `◇` empty state, 검색 결과 0건이지만 피드 자체는 비어있지 않을 때 `⌕` empty state (A13 3종)
- [ ] 툴바에 토픽명(또는 "전체") 표시, 메시지 수신 중일 때만 msg/s 레이트 표시(5초 창, F4), 수신 없으면 숨김
- [ ] 기록 중인 토픽 선택 시 기록 뱃지 + Live/Recorded 세그먼트 노출(G3), 기록 아닌 토픽에는 세그먼트 자체가 없음
- [ ] ⌕ 버튼 클릭 → 검색 행 표시, 입력 시 라이브 필터(payload substring, 전체 뷰에선 토픽명도 포함, 대소문자 무시) + N/M 매치 카운터 갱신(C26/C27)
- [ ] 검색 행에서 **Esc** → 검색 닫힘 + 쿼리 초기화(F28/F9), ✕ 버튼도 동일하게 동작
- [ ] 일시정지 버튼 → 목록이 그 순간의 스냅샷으로 고정되지만 수신 자체(및 msg/s)는 계속 진행됨(F24); 재개 시 최신 상태로 갱신
- [ ] 지우기 버튼 → 토픽 선택 시 해당 토픽만, 전체 뷰에서는 전체 메시지를 화면에서 제거(백엔드 History/Recorded 데이터는 보존, 표시만 클리어 — F3)
- [ ] 설정의 메시지 정렬을 "오래된 먼저"로 바꾸면 목록 순서가 즉시 뒤집힘(D61)
- [ ] 설정의 타임스탬프를 "상대 시각"으로 바꾸면 각 행 시각이 "N초 전"/"Ns ago" 등으로 표시되고 1초마다 자동 갱신됨(F25)
- [ ] retained 배지(R)와 QoS 배지에 마우스 오버 시 각각 tooltip 표시
- [ ] Recorded 뷰에서는 일시정지/지우기 버튼 대신 새로고침 버튼만 노출(G3), 클릭 시 최신 기록을 다시 로드
- [ ] Recorded 뷰를 보던 중 해당 토픽의 기록이 꺼지거나 토픽이 변경되면 자동으로 Live로 복귀함(G13 stuck 가드)

## 8. 메시지 상세 (B32~B36)

- [ ] 메시지 클릭 → 상세 패널에 topic/시각(절대, HH:MM:SS)/qos/바이트 크기 표시
- [ ] 포맷 탭(JSON/Plain/Hex/Base64) 전환 시 다른 메시지를 선택해도 포맷이 유지됨(sticky, G16 — 이전엔 메시지마다 자동 재감지였음)
- [ ] Diff 버튼 클릭 → 켜지면 포맷이 강제로 JSON으로 전환됨(C33)
- [ ] Diff on && 현재·직전 메시지가 모두 JSON 객체일 때: 변경된 키(노란 배경 + `← 이전값`), 추가된 키(초록 배경), 삭제된 키(빨강 배경 + 취소선), 변경 없는 키는 일반 표시(B36)
- [ ] Diff on이지만 비교 불가(직전 메시지 없음/배열/비-JSON)일 때 → 일반 페이로드 렌더로 폴백하되 Diff 버튼은 계속 활성 표시 유지(F15)
- [ ] content-type/response-topic/user property가 있는 메시지(5.0 연결) → 한 줄로 " · " 구분 조인되어 표시됨(G17, 이전엔 여러 줄이었음)

## 차트 (v2 인사이트)
사전: `docker exec mosq mosquitto_pub -t s/j -m '{"temp":23.4,"hum":61}'` 반복 + `mosquitto_pub -t s/n -m 23.4` 반복
- [ ] JSON 토픽 선택 → 상세 "차트" 탭 → temp/hum 키 칩 표시, 첫 키만 활성
- [ ] 칩 토글로 미니 차트 추가/제거 — 각 차트 자체 y스케일, 키 색 고정(토글해도 불변)
- [ ] 미니 차트 2개 이상일 때 hover 크로스헤어 동기화 + 툴팁(시각·값)
- [ ] 헤더 스탯 now/min/max/avg 실시간 갱신
- [ ] plain 숫자 토픽(s/n) → `value` 단일 키 차트
- [ ] 문자열 payload 섞인 메시지 → 차트에 갭(선 끊김)
- [ ] 숫자 없는 토픽 → "숫자 데이터가 없어요" empty state
- [ ] 일시정지 → 차트도 스냅샷 고정, 재개 시 갱신
- [ ] 기록 토픽에서 Recorded 전환 → 긴 히스토리 차트, Refresh 동작
- [ ] 설정 라이트 테마 → 차트 시리즈 색이 라이트 팔레트로 전환
- [ ] 모드 탭이 토픽 바꿔도 유지(sticky), 메시지 탭 복귀 시 포맷/Diff 정상
- [ ] 토픽 선택 시 툴바 msg/s가 해당 토픽 기준(전체 뷰 대비 값 감소 확인)

## 9. 발행 패널 (B37~B42)

- [ ] topic이 비어있거나 연결되어 있지 않으면 발행 버튼 비활성화(G8/C18)
- [ ] 트리에서 "이 토픽에 발행" 선택 또는 토픽 클릭 시 topic 입력이 자동으로 채워지고 힌트 텍스트가 뜨며, 직접 수정하면 힌트가 사라짐(C22/C24/C37)
- [ ] "속성" 토글 클릭 → 패널 높이가 176px→316px로 확장(애니메이션), 채워진 필드 수가 `· N`으로 표시됨
- [ ] MQTT 3.1.1 프로필로 연결된 상태에서는 속성 토글이 비활성화(opacity .45)되고 안내 카피가 노출됨(B40) — 5.0으로 재연결하면 다시 활성화
- [ ] content-type placeholder가 로컬라이즈되어 표시됨(F34)
- [ ] 발행 성공 후 자동으로 해당 토픽이 선택되고 방금 발행한 메시지가 상세에 표시됨(F2)
- [ ] 일시정지 상태에서도 발행 버튼을 누르면 정상적으로 발행됨(F32, 표시만 멈춰 있을 뿐 발행 자체는 항상 동작)

## 10. Recorded 뷰

- [ ] 특정 토픽 기록 켠 뒤 메시지 여러 개 발행 → 해당 토픽에서 Live/Recorded 세그먼트 노출
- [ ] Recorded로 전환 → 기록된 메시지가 시간순으로 표시, "새로고침" 클릭 시 최신 기록 재조회
- [ ] 기록을 켜지 않은 토픽에는 세그먼트 자체가 보이지 않음
- [ ] 앱을 완전히 재시작해도 기록 중이던 토픽에 트리 상의 ● 표시가 유지됨(RecordedTopics 초기화, G10)

## 11. 설정 모달 (A5, B53~B57)

- [ ] 타이틀바 ⚙ → 설정 모달 오픈(어느 뷰에서든), 헤더는 스크롤해도 sticky
- [ ] **모든 항목이 변경 즉시 적용됨**(C38/G6), 아래 각각을 바꾼 뒤 모달을 닫지 않고도 바깥 UI에 반영되는지 확인:
  - [ ] 언어(한국어/English) 전환 → 전체 UI 카피가 즉시 바뀜(재시작 불필요)
  - [ ] 테마 다크/라이트 전환 → 즉시 배색 전환
  - [ ] 테마를 "시스템"으로 설정 → OS 다크/라이트 모드를 바꾸면(매크OS 설정 또는 `Cmd+Ctrl+Q` 단축키 등) 앱이 즉시 따라감(matchMedia change 리스너)
  - [ ] 기본 페이로드 형식 변경 → 현재 열려 있는 메시지 상세의 포맷 탭도 즉시 전환됨(F7)
  - [ ] 타임스탬프(절대/상대), 메시지 정렬(최신/오래된) → 메시지 목록에 즉시 반영
  - [ ] 버퍼 슬라이더(50~500, step 10) 이동 → 현재값이 mono 숫자로 즉시 갱신되고, 백엔드 SetCapacity가 즉시 적용됨(G7/C39) — 슬라이더를 낮췄을 때 토픽별 보관 메시지 수가 즉시 트리밍되는지 확인
- [ ] ✕ 버튼, "완료" 버튼, 백드롭 클릭 모두 모달을 닫음(F7)
- [ ] **Esc**로도 모달이 닫힘(F28/C42)
- [ ] 앱을 재시작해도 위 설정값(및 언어/테마/힌트 노출 여부)이 그대로 유지됨(config 영속)

## 12. 키보드 지원 종합 (F28)

- [ ] Enter 제출: 연결 모달의 호스트/포트 입력, 구독 인라인 추가 입력, 트리 empty state의 특정 토픽 입력 — 3곳 모두 Enter로 제출됨
- [ ] Esc 닫기: 연결 모달, 설정 모달, `?` 가이드 오버레이, 메시지 검색 행 — 4곳 모두 Esc로 닫힘/취소됨
- [ ] 컨텍스트 메뉴는 기존과 동일하게 Esc로 닫힘(G14, 회귀 없음 확인)
